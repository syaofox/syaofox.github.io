Timeshift 的设计哲学是作为 **“系统还原”** 工具，而不是全面的数据备份工具。它的核心目标是保护**系统文件**和**设置**，让你在系统出问题时能快速恢复到一个正常的运行状态，同时**避免覆盖你的个人数据**（如文档、照片、音乐等）。

理解 Timeshift 的备份选项，尤其是**筛选器**，主要围绕以下几个关键点：

* * *

## 1\. 备份范围：系统 VS 用户数据

### 默认设置（核心设计）

Timeshift **默认只备份**你的**系统文件**和**系统配置**，包括：

+   `/`（根目录）下的大多数内容。
    
+   `/etc`（系统配置文件）。
    
+   `/boot`（启动文件）。
    
+   已安装的**程序文件**。
    

Timeshift **默认排除**（不备份）以下目录，它们主要存放用户数据：

+   `/home` 下的用户目录（例如 `/home/你的用户名/`）。
    
+   `/root`（root 用户的家目录）。
    
+   `/dev`, `/proc`, `/sys`, `/mnt`, `/media`, `/run`, `/tmp` 等临时或虚拟文件系统。
    

为什么排除用户数据？

当你还原快照时，Timeshift 会用快照中的文件覆盖当前系统中的对应文件。如果你的个人文件（如文档、照片）也被备份了，那么当你还原系统时，自创建快照后对这些个人文件所做的所有更改都会丢失。默认排除用户数据正是为了防止这种意外的数据丢失。

* * *

## 2\. 备份选项（设置）

### A. 快照类型 (Snapshot Type)

你需要在 Rsync 和 Btrfs 之间选择一种快照模式：

+   **Rsync (推荐)**：使用 **rsync** 工具和**硬链接 (hard-links)** 来创建快照。
    
    +   **优点**：兼容性最好，适用于几乎所有 Linux 文件系统（如 Ext4, XFS）。每个快照在文件管理器中都是一个**完整的、可浏览的系统副本**。只有在快照之间发生变化的文件才会被实际复制，未变动的文件通过硬链接指向前一个快照，从而节省空间。
        
+   **Btrfs**：仅适用于你的根分区 `/` 使用 **Btrfs 文件系统**，且采用特定的 **Ubuntu-style 子卷布局**（即有 `@` 和 `@home` 子卷）时。
    
    +   **优点**：速度极快，空间效率高，因为它使用的是 Btrfs 文件系统**原生的快照功能**。
        

### B. 计划任务 (Schedule)

你可以设置自动创建快照的频率和保留数量：

+   **Hourly (每小时)**
    
+   **Daily (每天)**
    
+   **Weekly (每周)**
    
+   **Monthly (每月)**
    
+   Boot (启动时)
    
    对于每种级别，你可以指定要保留多少个快照。例如，设置保留 5 个“每日快照”。
    

### C. 用户目录 (Users)

这是 Timeshift 备份选项中最关键的部分之一。它允许你决定如何处理用户的 Home 目录（`/home/你的用户名`）：

+   **Exclude All Files (默认)**：排除 Home 目录中的所有文件。**推荐保留此默认设置。**
    
+   **Include Hidden Files**：仅包含 Home 目录中的**隐藏文件和文件夹**（以 `.` 开头的文件/夹，通常包含应用配置、用户配置等）。如果你希望在还原系统时也能保留特定应用程序的用户级设置，可以考虑此选项。
    
+   **Include All Files**：包含 Home 目录中的**所有文件**。**通常不推荐**，除非你清楚知道后果（还原系统时会丢失自快照创建以来对个人文件的所有更改）。
    

* * *

## 3\. 筛选器 (Filters)

筛选器在设置中的 **“Filters (筛选器/过滤)”** 标签页下，它提供了对备份内容进行**精细控制**的能力。

筛选器列表由一系列路径或模式组成，每条规则前面有一个符号：

+   **`+` (Include/包含)**：明确**包含**这个文件、文件夹或模式。
    
+   **`-` (Exclude/排除)**：明确**排除**这个文件、文件夹或模式。
    

**注意：** 筛选器的规则是**自上而下**处理的。一旦一个文件/文件夹匹配了一个规则，就会执行该规则（包含或排除），后面的规则将不再对它生效。

### 常见的默认筛选规则

默认情况下，Timeshift 已经为你设置了一系列规则，例如：

1.  **`+ /`**：**包含**根目录 `/` 下的所有内容。
    
2.  **`- /home`**：**排除**整个 `/home` 目录。
    
3.  **`+ /home/user_name/`**：**（如果选择了包含隐藏文件或所有文件，会添加）** 包含用户的家目录。
    
4.  **`- /home/user_name/Documents/**`：**（如果选择了包含所有文件，你可能想添加此条）\*\* 排除用户文档文件夹中的所有内容。
    

### 如何使用筛选器？

你可以通过添加自己的规则来自定义备份范围：

| 目的 | 操作 | 示例 | 解释 |
| :--- | :--- | :--- | :--- |
| **排除**特定文件夹 | 添加一条 **- 规则** | `- /home/username/Downloads` | 排除下载文件夹。 |
| **包含**特定文件夹 | 添加一条 **+ 规则** | `+ /srv/data` | 包含 `/srv/data` 目录（例如服务器数据）。 |
| **排除**所有媒体文件 | 添加一个模式 **-** | `- \*.mp4, - \*.mp3` | 排除所有 `.mp4` 和 `.mp3` 文件。 |
| **Rsync 通配符** | | `*`，`**`，`***` | `*`：匹配目录中的任意字符（不含斜线）。`**`：匹配目录中或其下的任意字符（含斜线）。`***`：匹配目录本身及其下所有内容（如：`/home/user/cache/***`）。 |

**重要提醒：**

+   Timeshift 强烈建议你使用 **另一个工具**（如 **BackInTime** 或 **borg** 等）来备份你的个人数据（`/home` 目录）。
    
+   Timeshift 的快照应保存到**系统分区以外**的设备或分区上，以防主硬盘损坏时数据丢失。
    

**总结来说，Timeshift 专注于系统安全，筛选器和用户目录选项为你提供了一个明确的分界线：系统文件与个人数据应分开备份，Timeshift 只需照顾好你的操作系统。**
